Package development

One way of developing R packages is using the devtools, and usethis package https://cran.r-project.org/web/packages/usethis/usethis.pdf. R folder, man folder, DESCRIPTION file and NAMESPACE file are the minimum required content to build a package.

  1. create a project in a package-name new folder in Rstudio

  2. under the rstudio project, create a r file that will bag functions for your packages

  usethis::use_r('RfileName')
  1. automate documentation after insert roxygen skeleton into each function (code->Insert Roxygen Skeleton in rstudio IDE or Ctrl + shift + Alt + R on keyboard)
devtools::document()
  1. If your package includes small dataset, use use_data() to save existing data from the memory (called df, for example) into a data directory (It will create a directory if not available yet)
devtools::use_data(df,overwrite = TRUE)
  1. If you need to write R-script to generate your data for your package, your Rscript should be save in data_raw folder. Documentation for internal data detail skipped here
devtools::use_data_raw()
  1. create a rmarkdown file for automate md file for github (if you use github as your package repository)
devtools::use_readme_rmd()
  1. refresh all (r) files in the package before testing all functions within the package
devtools::load_all('.')
  1. install the package from the source (package could be saved in github as well)
devtools::install("your local directory/PackageName") or 
devtools::install_github("your github user name/PackageName")
